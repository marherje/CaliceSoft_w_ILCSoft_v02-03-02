Detailed Description

decode IDs with encoding string

This class decodes a single value from a CELL ID. In differs to the LCIO::BitField in two ways.

interpretation of Y-n
If the encoding string contains a variable name in the form of , is interpreted as offset to variable . It will be subsracted automatically. Example: If encodingstring is "K-1:3,I:3,J:3" and variable name is "K", will be returned.
no string comparison in decoding
All necessary string comparison is done during the initialisation. This should give some performance increase, if the same decoder is used many times.

The constructor is private. Use the FastDecoder::generateDecoder() function to get an object.
